【仕様概要(Web版)】
■ システム概要
  - 対象: モダンブラウザ（スマートフォン・PC対応）
  - 開発言語: HTML5 / CSS3 / JavaScript (ES Modules)
  - データ永続化: LocalStorage
  - GitHub Pages 対応（サーバーレス）

■ 画面構成
  - ローディング画面: 全アセット読み込み
  - タイトル画面: ケンバーンズエフェクト背景
  - 訓練生登録: 名前入力 + 先生名カスタマイズ
  - ストーリー画面: キャラクター会話（タイピングアニメ）
  - クイズ画面: 5科目 × 5問（3択・シャッフル対応）／ 2ゲームモード対応
  - エンディング画面: GOOD END / NORMAL END（パーティクル加飾）
  - 開発チーム画面: 本画面

■ 2ゲームモード
  Mode1: 乙女ゲームモード（assets/images/　← 先生リアル絵）
  Mode2: SDゲームモード  （assets/images_sd/ ← SDキャラ絵）
  ※ ローディング完了後に選択、タイトル画面からも再選択可能

■ 5科目 クイズ
  Network / PLC / Database / Java / Android
  各科目 約100問から5問をランダム出題

■ 親密度システム
  正解：+1 ／ 不正解：-1
  最高親密度 ≥ 3 → GOOD END

■ BGM・SE
  BGM: SUNO V5
  SE: 効果音ラボ様　提供素材

■ 解像度対応
  基準解像度: 1920×1080
  スマートフォン・PC のブラウザサイズに
  合わせてスケーリング対応

【設計仕様書】
■ アーキテクチャ
  - 構成: サーバーレス静的Webアプリ（GitHub Pages対応）
  - 言語: HTML5 / CSS3 / Vanilla JavaScript（ES Modules）
  - フレームワーク: 不使用（依存ライブラリゼロ）
  - モジュール構成:
      app.js          ← メインコントローラー（画面遷移・スケーリング）
      sound.js        ← SoundManager（BGM/SE/WakeLock）
      storage.js      ← LocalStorageラッパー
      confetti.js     ← 紙吹雪パーティクルエンジン（Canvas）
      screens/*.js    ← 各画面クラス（9画面）

■ 画面遷移設計
  Loading
    └→ Title
         ├→ NameInput → Story → Quiz ──→ Story（次ステージ）
         │                         └→ Ending
         ├→ Developer（開発チーム）
         ├→ QuizEditor（問題閲覧）
         └→ MusicTest（音楽テスト）

■ データ設計（LocalStorage）
  キー: "PolytechMemorial"
  構造:
    playerName    : 訓練生名（文字列）
    teacherNames  : 先生名配列（5要素）
    intimacy      : 親密度マップ { 1〜5: 数値 }
    progress      : ステージ進捗 { Network/PLC/..: {cleared} }
    allCleared    : 全クリアフラグ（真偽値）

■ メッセージシステム（外部JSON化）
  assets/data/messages_quiz.json       ← クイズメッセージ（4種×5先生）
  assets/data/messages_quiz_clear.json ← ステージクリア時メッセージ
  assets/data/messages_story.json      ← イントロ会話（5科目×5パターン）
  assets/data/messages_ending.json     ← エンディングメッセージ（5先生×5パターン）
  assets/data/teacher_names.json       ← 先生名シャッフル候補（50件）
  ※ 初回アクセス時のみfetch、以降はインスタンスキャッシュで管理

■ クイズシステム
  - 科目ごとにJSONから最大100問をロード
  - Fisher-Yatesシャッフルで5問を選択
  - 選択肢（正解1 + 不正解2）もシャッフル
  - 正解：親密度+1 / 不正解：親密度-1（最小0）
  - 親密度はステージクリア時のみLocalStorageへ確定保存
    （クリア前に戻るとロールバック）

■ エンディング判定
  GOOD END : いずれかの先生の親密度 ≥ 3
  NORMAL END: 全先生の親密度 < 3
  ※ タイトルへ戻ってもデータはリセットされない

■ スケーリング設計
  基準解像度: 1920 × 1080（固定）
  方式: CSS transform: scale() によるレターボックス表示
  対応: ウィンドウリサイズ・画面回転イベントで随時再計算

■ iOS PWA対応
  apple-mobile-web-app-capable : ホーム画面追加でフルスクリーン起動
  apple-touch-icon (180×180px) : ホーム画面アイコン設定済み
  ボタン操作時に requestFullscreen() を実行（PC除く・全画面への切替）

■ 音声設計
  BGM : HTMLAudioElement によるループ再生
        （Visibility API / WakeLock API 連携）
  SE  : Audio() の都度生成で多重再生対応
